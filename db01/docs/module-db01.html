Module: db01

MQTT → PostgreSQL subscriber microservice. Subscribes to the DR#1 topic tree on the MQTT broker, validates each incoming payload, and persists it to the appropriate PostgreSQL table (telemetry or equipment_status). Uses a connection pool so high message throughput doesn't exhaust connections.

Application entry point. Connects to the MQTT broker, subscribes to the DR#1 topic tree (wildcard under the configured namespace), routes every message to processMessageReceived, and installs pool error handling plus SIGINT/SIGTERM handlers that close the MQTT client and the pool on shutdown.

(async, inner) processMessageReceived(t, p) → {Promise.<void>}

Handles an incoming MQTT message: parses and validates the payload, derives the device and metric from the topic path, and inserts the reading into the correct table. Status topics (containing "status") go to equipment_status as a boolean; all other topics go to telemetry as a numeric value. Malformed payloads are rejected and logged rather than persisted.
