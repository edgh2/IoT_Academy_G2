Module: opc05

OPC-UA → MQTT publisher microservice for DR#1. Connects to the Beckhoff PLC's OPC-UA server, subscribes to the configured tags, and publishes each value change to the MQTT broker under the Magna namespace. This is the head of the pipeline — every downstream service depends on this feed.

(async, inner) handleDataReceived(tagname, dataValue, mqttclient) → {Promise.<void>}

Handles a received OPC-UA value by building the MQTT topic and publishing the reading. Status tags are routed under a "status/" sub-path. The topic is assembled from the configured namespace components (organization → workstation) plus the tag name, and the payload carries an ISO timestamp and the value.

Name Type Description

(async, inner) readOPCTags(tags) → {Promise.<void>}

Establishes the OPC-UA and MQTT connections and registers a subscription for every tag. Connects to the MQTT broker, connects to the OPC-UA server, creates a session and subscription, then wires up a monitored item per tag. Also installs SIGINT/SIGTERM handlers for graceful shutdown of both connections.
Parameters:
Name Type Description
tags Array.<Array.<string>> array of [tagTitle, tagNodeName] pairs to monitor.

(inner) setupTagSubscriptions(subscription, tagTitle, tag, mqttclient) → {void}

Creates an OPC-UA monitored item for a single tag and wires its "changed" event to the publish handler. Derives the device (e.g. Rob1, or "cell" if no Rob segment is present) from the tag name and prefixes the tag title with it, so published topics are grouped by device.
